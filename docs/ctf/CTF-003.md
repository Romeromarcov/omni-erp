# CTF-003: Shadow Mode Agents — Suite de Evaluación en CI

| Campo        | Valor                                           |
|--------------|-------------------------------------------------|
| **ID**       | CTF-003                                         |
| **Título**   | Eval suite para agentes en modo sombra          |
| **Regla**    | R-PROC-8 (agentes en producción requieren eval) |
| **Estado**   | ABIERTO                                         |
| **Fase**     | Fase 0 → Fase 1                                 |
| **Vence**    | 2026-09-01                                      |
| **Owner**    | equipo-agentes                                  |
| **ADR ref.** | ADR-004                                         |

## Deuda

Los agentes `AgenteReorden` y `AgenteCobranza` funcionan en modo sombra
(registran decisiones sin ejecutarlas). No existe una suite de evaluación
que mida la calidad de esas decisiones ni un pipeline de CI que bloquee
el merge si la precisión cae por debajo del umbral.

## Impacto

- No hay evidencia objetiva de que los agentes mejoran las decisiones.
- Promover un agente a modo autónomo sin eval es un riesgo operativo alto.

## Condición de cierre

1. Dataset de eval con ≥50 escenarios etiquetados por dominio
   (reorden: 25, cobranza: 25).
2. `pytest tests_eval/` ejecuta los escenarios y mide precision@1,
   recall y F1 por agente.
3. GitHub Actions ejecuta `tests_eval/` en cada PR que toca `apps/agentes/`.
4. Umbral: precision@1 ≥ 0.80 para pasar CI.
5. Dashboard Grafana muestra métricas históricas de sombra en producción.

## Riesgo de prórroga

Sin eval, la activación autónoma de agentes puede generar decisiones
incorrectas (órdenes de compra erróneas, mensajes de cobranza a clientes
al día) con impacto reputacional y económico directo.
